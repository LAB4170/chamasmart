# Plan 17.1: Backend Context Enforcement (IDOR Guards)

## Objective
Implement strict data isolation by enforcing multi-tenancy context across the backend. We must ensure that a user can only query, modify, or interact with records (members, cycles, transactions, loans) belonging to a Chama where they hold active membership. 

## Context
- .gsd/SPEC.md
- backend/middleware/auth.js (needs a `requireChamaMembership` guard)
- backend/controllers (various, need to enforce the guard)

## Tasks

<task type="auto">
  <name>Implement Chama Membership Middleware</name>
  <files>backend/middleware/auth.js</files>
  <action>
    - Create a new middleware function `verifyChamaMembership`
    - It should read `req.params.id` or `req.params.chamaId` (the Chama ID).
    - It must query the `chama_members` table to assert that `req.user.user_id` exists inside that specific `chama_id`.
    - If valid, attach the membership details (e.g., `req.chamaMembership`) and call `next()`.
    - If invalid or missing, return `403 Forbidden`.
  </action>
  <verify>grep -n "verifyChamaMembership" backend/middleware/auth.js</verify>
  <done>Middleware correctly intercepts requests and queries the DB for membership validation.</done>
</task>

<task type="auto">
  <name>Secure Chama Routes</name>
  <files>
    backend/routes/roscaRoutes.js
    backend/routes/welfareRoutes.js
    backend/routes/loanRoutes.js
  </files>
  <action>
    - Import `verifyChamaMembership` from the auth middleware.
    - Apply it to all routes that operate within the context of a specific Chama (e.g., creating cycles, applying for loans, fetching rosters).
  </action>
  <verify>grep -n "verifyChamaMembership" backend/routes/roscaRoutes.js</verify>
  <done>All multi-tenant API routes are protected against IDOR attacks by the membership guard.</done>
</task>

## Success Criteria
- [ ] Attempting to access `/api/chamas/:id/rosca/cycles` with a valid token but belonging to a different chama returns `403 Forbidden`.
- [ ] Legitimate members can still access their Chama's resources without disruption.
