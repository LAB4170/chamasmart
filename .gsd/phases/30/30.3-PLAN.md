# Plan 30.3: 24/7 AI Customer Support Bot

## Objective
Implement an autonomous AI customer support agent that listens on a dedicated 'Support' channel and responds contextually to members' messages using the Groq API.

## Context
- `.gsd/SPEC.md`
- `.gsd/phases/30/RESEARCH.md`
- `backend/controllers/chatController.js`

## Tasks

<task type="auto">
  <name>Setup AI Bot Identity</name>
  <files>
    - backend/scripts/seed_bot.js
  </files>
  <action>
    - Create a script to inject a static user into the database: `name='ChamaSmart AI Support', email='bot@chamasmart.com', role='SYSTEM'`.
    - Retrieve this Bot's ID to be used when emitting responses.
  </action>
  <verify>Run `seed_bot.js` and verify the bot user exists in the DB.</verify>
  <done>Bot user exists.</done>
</task>

<task type="auto">
  <name>Integrate LLM Support Logic</name>
  <files>
    - backend/controllers/chatController.js
    - backend/services/aiSupportService.js
  </files>
  <action>
    - In `chatController.js` (or a Socket listener), detect if a message is sent to a `type="support"` channel.
    - Create `aiSupportService.js` that takes the user's message, constructs a prompt (e.g., "You are ChamaSmart support. The user asked: ... Keep it brief and helpful."), and calls the Groq API (using the existing `groq-sdk`).
    - Once Groq responds, save the response to `chat_messages` under the Bot's User ID and emit `new_message` to the socket room.
  </action>
  <verify>Send a text to the support channel. Verify the Bot replies.</verify>
  <done>The AI accurately parses user questions and replies via the chat UI.</done>
</task>

## Success Criteria
- [ ] Any message sent in a 'support' channel triggers the Groq AI API.
- [ ] The AI responds properly in the chat flow as 'ChamaSmart AI Support'.
