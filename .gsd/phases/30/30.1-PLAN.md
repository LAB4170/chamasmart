# Plan 30.1: Backend Messaging Infrastructure

## Objective
Establish the database schema and Node.js backend architecture to support real-time messaging, including specific channels (groups vs. support) and `Socket.io` event handling for instant delivery.

## Context
- `.gsd/SPEC.md`
- `.gsd/phases/30/RESEARCH.md`
- `backend/socket.js`
- `backend/server.js`

## Tasks

<task type="auto">
  <name>Create Chat Database Schema</name>
  <files>
    - backend/scripts/migrate_chat.sql
    - backend/scripts/apply_chat_schema.js
  </files>
  <action>
    - Create a SQL migration script for `chat_channels` (id, chama_id, name, type: 'group' | 'support', created_at).
    - Create a SQL migration script for `chat_messages` (id, channel_id, user_id, message_type: 'text' | 'image' | 'video', content, media_url, created_at).
    - Ensure foreign keys correctly reference `chamas` and `users`.
  </action>
  <verify>Run `node backend/scripts/apply_chat_schema.js` and verify tables exist in PostgreSQL.</verify>
  <done>Tables `chat_channels` and `chat_messages` exist with correct constraints.</done>
</task>

<task type="auto">
  <name>Implement Chat REST API & Socket Handlers</name>
  <files>
    - backend/controllers/chatController.js
    - backend/routes/chat.js
    - backend/socket.js
    - backend/server.js
  </files>
  <action>
    - Create `chatController.js` with `getChannels`, `getMessages`, and `sendMessage`.
    - Update `socket.js` to handle `join_chat_channel` (socket.join) and `send_chat_message`.
    - When a message is sent via REST or Socket, save it to `chat_messages` and `io.to(channel).emit('new_message', data)`.
  </action>
  <verify>Start the server. Expect no crash. POST a message via cURL and expect a 201 response.</verify>
  <done>Backend can successfully store and emit chat messages.</done>
</task>

## Success Criteria
- [ ] Database schema for messaging is applied.
- [ ] REST API endpoints for chat history are functional.
- [ ] Socket.io emits message events to specific rooms.
